# Adjacency List (список смежности)

## Идея

Каждая строка хранит ссылку на своего родителя через `parent_id`. Самый простой
подход — схема понятна любому разработчику без объяснений.

## DDL

```sql
CREATE TABLE categories (
    id        bigserial PRIMARY KEY,
    name      text      NOT NULL,
    parent_id bigint    REFERENCES categories(id)
);
```

## Данные

```sql
INSERT INTO categories (id, name, parent_id) VALUES
(1, 'Продукты банка',    NULL),
(2, 'Кредиты',          1),
(3, 'Депозиты',         1),
(4, 'Потребительские',  2),
(5, 'Ипотека',          2),
(6, 'Вклады',           3),
(7, 'Накопительные',    3),
(8, 'Наличными',        4);
```

Структура дерева:

```
Продукты банка
├── Кредиты
│   ├── Потребительские
│   │   └── Наличными
│   └── Ипотека
└── Депозиты
    ├── Вклады
    └── Накопительные
```

## Запросы

### Прямые дети узла

Тривиальный запрос — просто фильтр по `parent_id`:

```sql
SELECT id, name
FROM categories
WHERE parent_id = 2;
```

Результат:

```
+--+----------------+
|id|name            |
+--+----------------+
|4 |Потребительские |
|5 |Ипотека         |
+--+----------------+
```

### Всё дерево — рекурсивный CTE

CTE (Common Table Expression, общее табличное выражение) — именованный подзапрос,
определяемый через `WITH`. Рекурсивный CTE ссылается сам на себя и выполняется
итерационно — пока не вернёт пустой результат.

```sql
WITH RECURSIVE tree AS (

    SELECT id, name, parent_id,
           0          AS depth,
           name::text AS path
    FROM categories
    WHERE parent_id IS NULL

    UNION ALL

    SELECT c.id, c.name, c.parent_id,
           t.depth + 1,
           t.path || ' > ' || c.name
    FROM categories c
    JOIN tree t ON c.parent_id = t.id

)
SELECT depth, path
FROM tree
ORDER BY path;
```

#### Как работает рекурсия пошагово

Запрос состоит из двух частей, соединённых `UNION ALL`. Это не просто объединение
двух результатов — в рекурсивном CTE `UNION ALL` означает: «выполняй нижнюю часть
снова и снова, пока она возвращает строки».

**Шаг 1 — якорь выполняется один раз**

Верхняя часть (до `UNION ALL`) — якорь. Возвращает корень дерева — строку
где `parent_id IS NULL`. Это стартовая точка. Временная таблица `tree` содержит:

```
id=1 | 'Продукты банка' | parent_id=NULL | depth=0 | path='Продукты банка'
```

**Шаг 2 — рекурсивная часть, первая итерация**

Нижняя часть (после `UNION ALL`) джойнит `categories` с `tree`. Здесь `tree` —
это результат **предыдущей итерации**. На первом шаге в `tree` лежит только
корень (id=1). Строка `JOIN tree t ON c.parent_id = t.id` — это и есть рекурсия:
`tree` ссылается сама на себя внутри своего определения.

Джойн находит строки где `parent_id = 1` — «Кредиты» и «Депозиты»:

```
id=2 | 'Кредиты'  | parent_id=1 | depth=1 | path='Продукты банка > Кредиты'
id=3 | 'Депозиты' | parent_id=1 | depth=1 | path='Продукты банка > Депозиты'
```

**Шаг 3 — рекурсивная часть, вторая итерация**

`tree` теперь содержит только строки из предыдущей итерации — id=2 и id=3.
Джойн находит строки где `parent_id IN (2, 3)`:

```
id=4 | 'Потребительские' | parent_id=2 | depth=2
id=5 | 'Ипотека'         | parent_id=2 | depth=2
id=6 | 'Вклады'          | parent_id=3 | depth=2
id=7 | 'Накопительные'   | parent_id=3 | depth=2
```

**Шаг 4 — рекурсивная часть, третья итерация**

`tree` содержит id=4,5,6,7. Джойн находит `parent_id IN (4,5,6,7)` — только «Наличными»:

```
id=8 | 'Наличными' | parent_id=4 | depth=3
```

**Шаг 5 — остановка**

`tree` содержит только id=8. Джойн ищет строки где `parent_id = 8` — таких нет.
Рекурсивная часть вернула пустой результат. PostgreSQL останавливается.

PostgreSQL накапливает финальный результат отдельно — добавляя к нему вывод
каждой итерации. На каждой итерации `tree` содержит только строки добавленные
на **предыдущей** итерации, не все накопленные.

#### Результат

```
+-----+---------------------------------------------------------+
|depth|path                                                     |
+-----+---------------------------------------------------------+
|0    |Продукты банка                                           |
|1    |Продукты банка > Депозиты                               |
|2    |Продукты банка > Депозиты > Вклады                      |
|2    |Продукты банка > Депозиты > Накопительные               |
|1    |Продукты банка > Кредиты                                |
|2    |Продукты банка > Кредиты > Ипотека                      |
|2    |Продукты банка > Кредиты > Потребительские              |
|3    |Продукты банка > Кредиты > Потребительские > Наличными  |
+-----+---------------------------------------------------------+
```

### Все потомки конкретного узла

```sql
WITH RECURSIVE subtree AS (

    SELECT id, name, parent_id, 0 AS depth
    FROM categories
    WHERE id = 2

    UNION ALL

    SELECT c.id, c.name, c.parent_id, s.depth + 1
    FROM categories c
    JOIN subtree s ON c.parent_id = s.id

)
SELECT id, name, depth
FROM subtree
WHERE id != 2
ORDER BY depth, name;
```

Результат:

```
+--+----------------+-----+
|id|name            |depth|
+--+----------------+-----+
|4 |Потребительские |1    |
|5 |Ипотека         |1    |
|8 |Наличными       |2    |
+--+----------------+-----+
```

### Все предки конкретного узла

```sql
WITH RECURSIVE ancestors AS (

    SELECT id, name, parent_id
    FROM categories
    WHERE id = 8

    UNION ALL

    SELECT c.id, c.name, c.parent_id
    FROM categories c
    JOIN ancestors a ON c.id = a.parent_id

)
SELECT id, name
FROM ancestors
WHERE id != 8
ORDER BY id;
```

Результат:

```
+--+----------------+
|id|name            |
+--+----------------+
|1 |Продукты банка  |
|2 |Кредиты         |
|4 |Потребительские |
+--+----------------+
```

### Вставка нового узла

Просто `INSERT` с указанием `parent_id`:

```sql
INSERT INTO categories (name, parent_id)
VALUES ('Рефинансирование', 5);
```

### Перемещение узла

Просто `UPDATE` поля `parent_id`. Всё поддерево переезжает автоматически —
его строки ссылаются на `id` перемещаемого узла, а не на старого родителя:

```sql
UPDATE categories
SET parent_id = 3
WHERE id = 4;
```

## Trade-offs

| | |
|---|---|
| Плюсы | Простая схема, лёгкая вставка и перемещение узлов |
| Минусы | Получение всего поддерева требует рекурсивного CTE — дорого на глубоких деревьях |
| Когда применять | Дерево активно меняется, глубина небольшая. Например: оргструктура банка |
